Single DB — data isolated by tenant foreign key

How multi-tenancy works: All tenants share one SQLite database. Every Customer and User row has a tenant_id foreign key. When you query, Django filters WHERE tenant_id = your_tenant automatically — so Tenant A never sees Tenant B's data.
